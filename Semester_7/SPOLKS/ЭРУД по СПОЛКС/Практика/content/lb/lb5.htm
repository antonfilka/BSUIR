Общие требования по выполнению лабораторных работ

Лабораторная работа № 5 "Сетевое системное программное обеспечение"

Тема работы: Реализация утилит ping и traceroute, изменение адреса источника и приемника.


Теоретические сведения: тема 11

Требования:

  • ОС Linux (последние версии Windows не позволяют отправлять пакеты с модифицированным адресом отправителя),

  • C/C++ либо другой язык позволяющий формировать поля IP заголовка (изменение TTL, задание адреса отправителя)

Реализация программы параллельного ping. Выполнить пинг нескольких хостов.

Время задержки передачи = (время приема - время отправки)/2

Возможные реализации:

  • В эхо запросе отправлять идентификатор потока сформировавшего запрос и ожидающего ответ. Каждый поток считывает из буфера входящие пакеты с флагом MSG_PEEK. Если идентификатор принятый в эхо-отклике совпадает с идентификатором текущего потока – выполняется повторное вычитывание без флага.

  • Метка времени отправки эхо-запроса посылается внутри пакета. Поток, получивший эхо-ответ вычисляет время затраченное на передачу.


Добавить возможность определения пути до каждого узла назначения (traceroute), путем модификации в цикле поля TTL IP заголовка.


Преобразовать утилиту ping для выполнения атаки Smurf (в качестве адреса источника указывается адрес атакуемого узла в заголовке IP­пакета). Продемонстрировать на атакуемом узле по средствам Wireshark.
